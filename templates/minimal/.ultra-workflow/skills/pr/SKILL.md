# pr

## Template

- Problem
- Decision
- Implementation Summary
- Test Evidence
- Risks and Rollback

## Rules

- Keep PR focused to one coherent objective.
- Include commands or CI links proving validation.
- Call out backward-incompatible changes explicitly.
