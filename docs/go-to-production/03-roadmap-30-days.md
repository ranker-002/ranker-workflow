# 30-Day Roadmap

## Week 1: Stabilize

- Freeze command surface for current release branch.
- Run CLI tests on Linux/macOS.
- Validate all templates and CI provider files.
- Publish release candidate internally.

## Week 2: Pilot

- Onboard 3 real repositories.
- Track KPIs: LTG, FPGR, PMDR.
- Collect friction notes from developers.
- Ship small fixes only (no large scope changes).

## Week 3: Harden

- Address top 5 friction points from pilot.
- Improve failure guidance for most frequent errors.
- Validate migrate flow on older scaffold snapshots.
- Draft public launch docs and examples.

## Week 4: Launch

- Publish stable npm release.
- Publish GitHub release and migration notes.
- Announce OSS launch with benchmark data.
- Start weekly KPI reporting cadence.
