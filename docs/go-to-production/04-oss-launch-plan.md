# OSS Launch Plan

## Positioning

- Promise: reliable feature delivery with strict gates and low process overhead.
- Audience: teams using AI coding agents with mixed model quality.

## Launch Assets

- One-page README value proposition.
- Quickstart video or GIF (init -> feature -> run -> status).
- Comparison table against alternative workflows.
- Public benchmark report from pilot repos.

## Launch Sequence

1. Publish npm package and GitHub release.
2. Publish benchmark + KPI baseline.
3. Post launch thread with install command and quickstart.
4. Open contributor docs and roadmap issues.

## Community Operations

- Weekly triage for issues.
- Public changelog and migration notes each release.
- Monthly KPI summary for transparency.

## Competitive Moat

- Keep reliability data public and updated.
- Expand deterministic checks and migration safety.
- Optimize UX based on measured friction, not assumptions.
